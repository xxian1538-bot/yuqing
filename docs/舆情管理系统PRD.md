## 2. 产品背景

当前系统面向政企单位的网络舆情发现、研判、报送、处置、网评、审核与留痕流程。系统已围绕“舆情事件 -> 任务指派 -> 任务执行 -> 审核 -> 完结”的主流程建立功能闭环，并支持历史案例参考、事件关联、评分评级和工作流配置。

系统目标是将分散的舆情事件、处置动作、网评动作和审核记录统一管理，帮助舆情管理员、处置人员、网评人员、审核人员和领导角色在一个平台内完成协同处置。

## 3. 产品目标

1. 建立舆情事件统一台账，支持新增、编辑、查询、详情查看、关联和手动完结。
2. 支持舆情事件按评分权重自动计算等级，且权重变更仅影响后续新增事件。
3. 支持舆情事件向人员、角色、组织、职位灵活指派处置任务或网评任务。
4. 支持处置任务、网评任务接收、执行、材料录入和提交审核。
5. 支持审核流程按场景配置，覆盖处置任务完结审核和网评任务完结审核。
6. 支持历史已完结舆情事件作为处置参考案例，任务详情中可跳转查看案例详情。
7. 支持舆情详情页以全景链路、事件脉络、处置记录、自动评价等方式查看处置过程。
8. 支持导出舆情详情为 Word 兼容文档。

## 6. 核心业务流程

### 6.1 舆情事件管理流程

1. 舆情管理员进入“舆情展示”。
2. 可通过搜索、状态、倾向等条件筛选舆情。
3. 可手动新增舆情事件，录入基础信息、互动量、评分维度。
4. 系统根据当前评分权重计算事件分数和等级。
5. 已有舆情事件支持编辑，编辑字段与新增字段保持一致。
6. 单条或多条舆情事件可进行关联，形成主事件和关联事件关系。
7. 舆情详情页展示基本信息、内容、互动数据、处置过程和事件脉络。
8. 舆情可报送、指派任务、导出 Word、手动完结。

### 6.2 任务指派流程

1. 舆情管理员在列表页或详情页点击“指派任务”。
2. 选择任务类型：处置任务或网评任务。
3. 选择处理对象，支持人员、角色、组织、职位，并支持搜索多选。
4. 可选择历史已完结舆情事件作为参考案例。
5. 创建任务后，舆情任务状态进入处置中。
6. 批量指派时，系统对选中的每条舆情分别生成对应任务。

### 6.3 处置任务流程

1. 处置任务创建后状态为“未接收”。
2. 处理人员在处置任务列表点击“接收”，状态变为“已接收”。
3. 接收后可点击“执行”，录入任务执行内容、完成时间、附件材料。
4. 选择处置任务完结审核流并提交审核。
5. 审核通过后，任务状态变为“已完结”。
6. 审核退回后，任务可继续补充材料并重新提交。

### 6.4 网评任务流程

1. 网评任务创建后状态为“未接收”。
2. 网评人员点击“接收”，状态变为“已接收”。
3. 接收后可点击“执行”，录入平台、账号、链接、内容、互动量和执行小结。
4. 可提交单条记录暂不完结，也可提交并申请完结任务。
5. 申请完结时需选择网评任务审核流。
6. 审核通过后，任务状态变为“已审核”。
7. 审核退回后，任务状态变为“未通过”。

### 6.5 审核流程

1. 系统支持固定两类审核场景：
   - 处置任务完结审核
   - 网评任务完结审核
2. 每个场景可配置多条审核流。
3. 每条审核流支持多级审核层级，默认三级，可新增或删除层级。
4. 每个审核层级可配置人员、角色、组织、职位作为审核对象。
5. 审核方式支持或签、会签。
6. 列表页支持设为默认、编辑、删除、启用/停用。
7. 当前执行态为单次通过/退回机制，多级逐级流转作为后续增强项。

### 6.6 事件完结流程

1. 当舆情下所有任务完成并审核通过后，任务状态进入“待完结”。
2. 管理员可在列表或详情页手动完结事件。
3. 领导确认完结时需填写完结说明。
4. 完结后舆情状态变为“已办结”，任务状态展示为“已完结”。
5. 完结记录展示在处置记录和“我发起的”页面。

## 9. 状态定义

### 9.1 舆情状态

| 状态 | 含义 |
| --- | --- |
| 未处理 | 事件已入库但未进入处置 |
| 跟进中 | 已指派任务或正在处置 |
| 已报送 | 事件已报送 |
| 已办结 | 事件已最终完结 |

### 9.2 舆情任务状态

| 状态 | 计算逻辑 |
| --- | --- |
| 待指派 | 未生成任何处置任务或网评任务 |
| 处置中 | 存在未完结或未审核通过任务 |
| 待完结 | 所有任务完成并审核通过，但事件未最终完结 |
| 已完结 | 舆情事件状态为已办结 |

### 9.3 处置任务状态

| 状态 | 含义 |
| --- | --- |
| 未接收 | 任务已下发，等待处理人员接收 |
| 已接收 | 任务已接收，待执行 |
| 处置中 | 正在执行 |
| 已完成 | 已提交完成材料 |
| 无法处置 | 无法执行或需异常处理 |
| 已完结 | 审核通过并完结 |

### 9.4 网评任务状态

| 状态 | 含义 |
| --- | --- |
| 未接收 | 任务已下发，等待网评人员接收 |
| 已接收 | 任务已接收，待执行 |
| 进行中 | 已提交部分执行记录 |
| 已提交 | 已申请完结审核 |
| 已审核 | 审核通过 |
| 未通过 | 审核退回 |

## 12. 验收标准

1. 舆情列表可正常加载、搜索、筛选、分页。
2. 新增舆情事件可录入完整字段并计算等级。
3. 编辑舆情事件字段与新增字段保持一致，保存后列表和详情同步更新。
4. 单条和批量关联舆情后，详情页可展示事件脉络。
5. 舆情详情页导出生成 Word 兼容文档。
6. 报送弹窗可提交报送说明并更新报送状态。
7. 处置任务和网评任务均必须先接收再执行。
8. 处置任务执行后可提交审核，通过后任务完结。
9. 网评任务执行后可提交审核，通过后任务状态为已审核。
10. 历史参考案例在任务详情中可点击进入舆情详情页。
11. 工作流配置支持多流程、默认流程、启用/停用、编辑、删除。
12. 系统配置中评分权重保存后只影响后续新增舆情。
13. 刷新页面后，MySQL 中已保存的数据仍可加载。

## 13. 后续规划

1. 增加真实用户登录、角色权限和操作审计。
2. 将审核执行从单次通过/退回升级为逐级流转。
3. 支持会签场景下多人分别审批和进度展示。
4. 支持附件真实上传、预览和下载。
5. 支持报送记录独立菜单和报送回执。
6. 支持舆情数据导入、批量删除和高级检索。
7. 支持统计分析报表在权限允许范围内恢复或独立配置。
8. 支持生产部署配置、数据库迁移脚本和初始化脚本。
